HyperFS is unified SAN & NAS storage file system, providing Fibre Channel & iSCSI SAN access and scale-out clustered NAS access at 1Gb/10Gb/50Gb Ethernet (such as Samba, NFS and FTP) all via a unified storage namespace. This unified file system can provide traditional block-level access to high-performance systems while also supporting hundreds of NAS clients all at the same time and with the same media. The cluster structure of the NAS Gateway ensures strong scalability with performance and capacity to meet the requirement of storage of mass unstructured data.

HyperFS Features

  • Integrated SAN & NAS under a single global Namespace
  • Support for up to 4093 LUNs (LUN may be single disk or full RAID array)
  • Full Mac OS X Yosemite support for XSAN Migrations
  • Support for billions of files/folders/objects
  • 64 ZB Theoretical File System Limit
  • 64 ZB Theoretical File Size Limit
  • Dynamic file system expansion
  • Full SSD Support
  • Cross Platform Client Support
    • Windows XP/Vista/Win7 32/Win 8 32/x86_64
    • Windows 2003/2008 Server/2012 Server 32/x86_64
    • RHEL 5(U3-U10) 32/x86_64
    • RHEL 6(U0-U6)32/x86_64
    • SUSE 11 SP1 32/x86_64
    • Mac OS X 10.6/10.7/10.8/10.9/10.10
  • Multiple Simultaneous Block Level Protocols
    • Fibre Chanel
    • iSCSI
    • Infiniband

Scale-Out NAS Features

  • Users can Scale-Out performance or Scale-Up storage capacity independently
  • Support for 64 Nodes per Cluster
  • Support for up to 32 Clusters
  • Each Node delivers 500MBs/ writes & 700MB/s reads
  • Supported Protocols
    • CIFS/SMB v1.0/v2.0/v3.0 - 2048 Clients per Node
    • NFS v2/v3/v4 - Unlimited Clients per Node
    • FTP/FTPs - 800 on 1Gb or 1600 on 10Gb Clients per Node
    • HTTP/HTTPS/WebDAV
  • Web-based admin tools makes adding or replacing (HA) nodes simple
